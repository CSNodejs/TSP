What is it?

  • server side
  • event driven
    (enters a main loop, waiting for an event to occur and after responding to the event, returns to the loop and waits for the next event)
  • javascript
...that runs on the obscenely fast V8 engine from Google.

http://code.google.com/p/v8/

What is it used for?

Here's an example of a very simple http server. But there are many more uses.
  • Static file servers
  • Web Application frameworks
  • Messaging middle ware
  • Servers for HTML5 multi player games

Getting started:

There are lots of ways to install.

  • OS X:
    $ brew install node*
    $ port install node*
  • Linux:
    $ apt-get install node
  • Windows/all: http://nodejs.org/#download
* You'll need Xcode or this GCC Installer for OS X

NVM: Node Version Manager

Node Version Manager is a tool for switching between releases of node. This is useful for testing your code before deployment, or testing your open source code against different versions of node. There have been API changes lately so itâ€™s worth having a sanity check against major releases.

NVM: Usage

  • Install:
    $ nvm install v0.6.12
  • Use:
    $ nvm use v0.6.12

NPM: Node Package Manager

NPM

This is a package manager that many people add to in order to make it very simple to install their code.

NPM: Usage

  • Install:
    $ curl http://npmjs.org/install.sh | sh (included by default with Node.JS, since v0.6.3)
  • Use:
    $ npm install socket.io
    (installs in ./node_modules) OR
    $ sudo npm install -g socket.io
    (installs globally, typically in ~/node_modules)
